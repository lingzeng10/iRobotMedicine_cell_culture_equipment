║           為什麼不同 WiFi 無法連線？網路基礎知識               ║
╚══════════════════════════════════════════════════════════════╝

📚 簡單解釋
═══════════════════════════════════════════════════════════════

區域網路 IP 地址（如 192.168.0.186）只能在同一個路由器下使用！

═══════════════════════════════════════════════════════════════

🏠 類比：把 WiFi 想像成一棟大樓
═══════════════════════════════════════════════════════════════

想像一下：

【情況 A：同一棟大樓】
───────────────────────────────────────────────────────────────
   您的電腦：在「大樓A」的「3樓306號房」
   朋友的手機：在「大樓A」的「5樓501號房」
   
   兩者都在同一棟大樓 → 可以互相訪問 ✅
   
   這就是「同一個 WiFi」的情況
   
   您的電腦 IP：192.168.0.186（大樓A的306號房）
   朋友可以訪問：192.168.0.186 ✅

═══════════════════════════════════════════════════════════════

【情況 B：不同的大樓】
───────────────────────────────────────────────────────────────
   您的電腦：在「大樓A」的「3樓306號房」
   朋友的手機：在「大樓B」的「2樓201號房」
   
   兩者在不同大樓 → 無法直接訪問 ❌
   
   這就是「不同 WiFi」的情況
   
   您的電腦 IP：192.168.0.186（大樓A的306號房）
   朋友的設備：在完全不同的網路（大樓B）
   
   朋友嘗試訪問：192.168.0.186
   → 找不到！因為不在同一棟大樓（不同路由器）❌

═══════════════════════════════════════════════════════════════

📡 技術解釋
═══════════════════════════════════════════════════════════════

1. 什麼是區域網路 IP（192.168.x.x）？
───────────────────────────────────────────────────────────────
   
   區域網路 IP（如 192.168.0.186）是路由器分配給連接設備的地址
   
   • 只在同一個路由器（WiFi）下有效
   • 不同路由器下可能會有相同的 IP（如兩個路由器都有 192.168.0.186）
   • 但彼此無法互相訪問，因為是不同的網路

═══════════════════════════════════════════════════════════════

2. 路由器（WiFi）如何工作？
───────────────────────────────────────────────────────────────
   
   每個路由器創建一個獨立的區域網路：
   
   【路由器 A】                    【路由器 B】
   ┌─────────────┐                ┌─────────────┐
   │  192.168.0.x│                │  192.168.1.x│
   │              │                │              │
   │ 您的電腦      │                │ 朋友的設備    │
   │ 192.168.0.186│                │ 192.168.1.50 │
   └─────────────┘                └─────────────┘
         ↓                              ↓
   無法互相訪問 ❌                  無法互相訪問 ❌
   
   即使兩個設備都有「192.168.0.186」這個地址
   但在不同路由器下，它們是完全不同的設備！

═══════════════════════════════════════════════════════════════

3. 為什麼需要同一個 WiFi？
───────────────────────────────────────────────────────────────
   
   當朋友的手機連接不同的 WiFi：
   
   1. 路由器分配不同的區域網路 IP
      • 您的電腦：192.168.0.186（路由器A）
      • 朋友手機：192.168.1.50（路由器B）
   
   2. 朋友嘗試訪問 192.168.0.186
      • 路由器的網路範圍只在 192.168.1.x（路由器B）
      • 找不到 192.168.0.186（因為這個地址在路由器A）
      • 結果：無法連線 ❌
   
   3. 即使同一個路由器，但連到「訪客網路」
      • 訪客網路被隔離（安全設定）
      • 無法訪問主網路的設備
      • 結果：無法連線 ❌

═══════════════════════════════════════════════════════════════

🌐 IP 地址的層級
═══════════════════════════════════════════════════════════════

【區域網路 IP（私有 IP）】
───────────────────────────────────────────────────────────────
   範圍：192.168.x.x、10.x.x.x、172.16.x.x - 172.31.x.x
   
   特點：
   • 只在同一個路由器（區域網路）內有效
   • 不同路由器可以有相同的區域網路 IP
   • 無法直接從網際網路訪問
   • 例如：192.168.0.186

═══════════════════════════════════════════════════════════════

【網際網路 IP（公有 IP）】
───────────────────────────────────────────────────────────────
   範圍：除了私有 IP 以外的所有地址
   
   特點：
   • 全球唯一（不會重複）
   • 可以從網際網路任何地方訪問
   • 例如：203.xx.xx.xx（您的路由器對外的 IP）
   
   注意：
   • 即使是公有 IP，從網際網路訪問也需要：
     - 路由器設定（端口轉發）
     - 防火牆允許
     - 通常需要額外設定

═══════════════════════════════════════════════════════════════

📋 實際範例
═══════════════════════════════════════════════════════════════

範例 1：同一 WiFi ✅
───────────────────────────────────────────────────────────────
   您的電腦：
   • 連接 WiFi：「MyOffice」
   • IP：192.168.0.186
   • 路由器：辦公室路由器
   
   朋友的手機：
   • 連接 WiFi：「MyOffice」
   • IP：192.168.0.50
   • 路由器：同一個（辦公室路由器）
   
   結果：可以互相訪問 ✅
   因為在同一個區域網路下

═══════════════════════════════════════════════════════════════

範例 2：不同 WiFi ❌
───────────────────────────────────────────────────────────────
   您的電腦：
   • 連接 WiFi：「MyOffice」
   • IP：192.168.0.186
   • 路由器：辦公室路由器
   
   朋友的手機：
   • 連接 WiFi：「HomeWiFi」
   • IP：192.168.1.50
   • 路由器：家裡的另一個路由器
   
   結果：無法互相訪問 ❌
   因為在不同的區域網路下

═══════════════════════════════════════════════════════════════

範例 3：同路由器但訪客網路 ❌
───────────────────────────────────────────────────────────────
   您的電腦：
   • 連接 WiFi：「MyOffice」
   • IP：192.168.0.186
   • 路由器：辦公室路由器
   
   朋友的手機：
   • 連接 WiFi：「MyOffice-Guest」（訪客網路）
   • IP：192.168.0.100
   • 路由器：同一個，但訪客網路被隔離
   
   結果：無法互相訪問 ❌
   因為訪客網路與主網路被隔離（安全設定）

═══════════════════════════════════════════════════════════════

範例 4：手機數據網路 ❌
───────────────────────────────────────────────────────────────
   您的電腦：
   • 連接 WiFi：「MyOffice」
   • IP：192.168.0.186（區域網路 IP）
   
   朋友的手機：
   • 關閉 WiFi，使用 4G/5G 數據網路
   • IP：由電信公司分配的公網 IP（如 111.xx.xx.xx）
   
   結果：無法直接訪問 ❌
   因為：
   • 區域網路 IP（192.168.x.x）無法從網際網路直接訪問
   • 需要通過路由器的公有 IP 和端口轉發（需要額外設定）

═══════════════════════════════════════════════════════════════

🔧 如何解決？
═══════════════════════════════════════════════════════════════

【最簡單的解決方法】
───────────────────────────────────────────────────────────────
   讓朋友的手機連接到與您相同的 WiFi
   
   步驟：
   1. 在朋友的手機上開啟 WiFi 設定
   2. 找到與您電腦相同的 WiFi 名稱
   3. 輸入密碼連接
   4. 確認連接成功
   5. 然後訪問：http://192.168.0.186:3000

═══════════════════════════════════════════════════════════════

【如果要從網際網路訪問（進階）】
───────────────────────────────────────────────────────────────
   如果您想讓朋友從任何地方訪問（不限制 WiFi）：
   
   需要設定：
   1. 路由器端口轉發（Port Forwarding）
      • 將外部端口轉發到您的電腦
      • 需要知道路由器的公有 IP
   
   2. 防火牆設定
      • 路由器防火牆允許連線
      • 電腦防火牆允許連線
   
   3. 使用動態 DNS（如果 IP 會變動）
      • 因為家用網路 IP 可能會改變
   
   注意：
   • 這需要網路管理權限
   • 可能有安全風險
   • 不建議用於臨時分享

═══════════════════════════════════════════════════════════════

💡 總結
═══════════════════════════════════════════════════════════════

簡單來說：
   • 區域網路 IP（192.168.x.x）只在同一個路由器下有效
   • 不同的 WiFi = 不同的路由器 = 不同的網路 = 無法直接訪問
   • 就像不同大樓的房間號碼，即使號碼相同也無法直接訪問
   
最簡單的解決方法：
   • 讓朋友連接到與您相同的 WiFi ✅
